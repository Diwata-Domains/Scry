# Security Policy

## Reporting a vulnerability

Email **security@diwata.domains** with a description of the issue and steps to reproduce. Do not open
a public issue for a vulnerability.

Include where relevant:

- the affected version or commit,
- a minimal reproduction,
- the impact you've assessed.

## What to expect

- Acknowledgment within 3 business days.
- An assessment and, if confirmed, a fix timeline.
- Credit in the release notes if you'd like it.

Please give a reasonable window to release a fix before any public disclosure. Coordinated
disclosure is appreciated.

## Scope

This policy covers **Scry Core** (this repository). Diwata's private editions and hosted
infrastructure are out of scope here — report those concerns to the same address and they will be
routed internally.
